ESLint v5.7.0 released

We just pushed ESLint v5.7.0, which is a minor release upgrade of ESLint. This release adds some new features and fixes several bugs found in the previous release.

Highlights

Three rules gained new options in this release:

  • padding-line-between-statements has a new iife node type to specifically target immediately-invoked function expression invocations as distinct statements.
  • no-tabs has a new allowIndentationTabs option to permit tabs only for indentation and nowhere else.
  • camelcase has a new allow option that can be configured to individually allow certain non-camelcase identifiers like React's UNSAFE_componentWillMount.

Enhancements

  • 6cb63fd Update: Add iife to padding-line-between-statements (fixes #10853) (#10916) (Kevin Partington)
  • 5fd1bda Update: no-tabs allowIndentationTabs option (fixes #10256) (#10925) (Kevin Partington)
  • 066f7e0 Update: camelcase rule ignoreList added (#10783) (Julien Martin)
